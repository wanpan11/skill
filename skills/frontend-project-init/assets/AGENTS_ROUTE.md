## 为什么有多个路由出口

路由出口用于渲染当前 URL 匹配到的路由组件。具体路由出口写法需要从目标项目读取。

如果项目使用嵌套路由，通常会有多层路由出口：

```text
[根组件] 的 [路由出口]
  -> 渲染第一层 [layout 或页面壳层]

[layout 或页面壳层] 内部的 [路由出口]
  -> 渲染第二层业务页面
```

如果删除 layout 内部的路由出口，外层页面壳层可能能显示，但业务页面没有出口，页面内容不会渲染。

## 路由出口 key 的含义

路由出口负责渲染当前匹配到的路由组件。

如果路由出口带有 `key`：

```text
[路由出口 key 写法]
```

表示当该 key 变化时，强制销毁并重新创建当前页面组件。

注意区分目标项目中的地址字段：

- `[不包含查询参数的地址字段]`：`[行为说明]`。
- `[包含查询参数的完整地址字段]`：`[行为说明]`。

如果只改变查询参数就需要重建页面，通常应检查 key 是否使用了包含查询参数的完整地址字段或等价能力。

## 常见排查入口

- 页面不渲染：先查路由是否匹配、layout 内部是否有路由出口、组件是否被正确挂载。
- 菜单不显示：先查菜单来源、权限配置、当前 layout、路由 meta。
- 页面不重建：先查路由出口 key 使用的地址字段是否包含查询参数。
- 页面壳层不对：先查路由配置字段、系统页面框架配置和 layout 选择逻辑。
- 权限或跳转异常：先查导航守卫、登录态、权限数据和路由白名单。
