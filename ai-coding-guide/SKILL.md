# AI 编码指南

## 用途

使用此 skill 为目标项目创建或刷新 `AGENTS.md`，让后续 AI agent 能快速理解项目事实并遵守统一编码规范。

此 skill 内置模板位于 `assets/AGENTS.md`。模板来自当前项目的 `AGENTS.md` 结构，保留通用 AI 编码规范，并把项目特定描述改成需要现场填充的内容。

## 工作流程

### 1. 读取目标项目

进入目标项目后，先读取真实上下文：

- 项目说明：`README.md`、`AGENTS.md`、`CLAUDE.md`、`CODEX.md`、任务相关文档。
- 包和命令：`package.json`、锁文件、构建配置、测试配置、lint/typecheck 配置。
- 代码结构：入口文件、根组件、路由、布局、状态管理、请求封装、工具目录、页面目录。
- 本地约束：环境变量、代理配置、登录态、token、Cookie、构建产物、不得提交的文件。
- 既有规范：目录 review 要求、专用 skill、提交信息格式、验证命令策略。

只填入目标项目中能确认的事实。无法确认的内容保留为“待补充”或向用户询问，不要编造。

### 2. 使用模板生成 `AGENTS.md`

复制 `assets/AGENTS.md` 到目标项目根目录的 `AGENTS.md`，然后按目标项目实际情况替换模板中的占位内容。

处理规则：

- 保留模板中的通用规范章节，尤其是“编码约定”“注释与文风约定”“协作原则”“提交前检查”“提交信息规范”“Karpathy 编码指南”。
- 用目标项目事实改写“项目概览”“常用命令”“本地启动注意事项”“目录约定”“参考文档”。
- 删除目标项目不适用的条目。
- 对高风险或未确认内容，写成明确的待确认说明，不要假装确定。
- 如果目标项目已有 `AGENTS.md`，先读原文件并合并，不要直接覆盖用户已有规则。

### 3. 给 AI 提供规范

生成后的 `AGENTS.md` 应同时服务两个目标：

- 快速初始化：让新进入项目的 AI agent 先读一份稳定、完整的项目协作说明。
- 行为约束：要求 AI agent 最小改动、动态读取项目事实、保护用户改动、避免过度抽象、按需验证。

### 4. 验证

完成后检查：

- `AGENTS.md` 不包含其他项目的项目名、端口、目录、依赖、负责人或敏感值。
- 所有项目特定描述都来自目标项目可见文件或用户确认。
- 通用规范章节没有被不必要地删减。
- 如果目标项目是 git 仓库，检查 `git status --short`，确认只有预期文件变更。

## 模板资源

- `assets/AGENTS.md`：可复制到目标项目的中文 `AGENTS.md` 模板。
